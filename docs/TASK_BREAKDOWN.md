## Appendix: Key Technical Decisions

1. **No DI framework**: Manual constructor injection at the API composition root. Simple and debuggable for MVP scope.
2. **Domain entities are NOT TypeORM entities**: Strict separation. Mappers convert between the two at the infrastructure boundary.
3. **All queries are clinic-scoped**: Multi-tenancy is enforced at the repository level, not just the API level.
4. **Event handling starts in-process**: Domain events are handled synchronously within the same request for audit logs. Emails and reminders are dispatched asynchronously via BullMQ.
5. **Redis serves triple duty**: Slot caching, rate limiting, and BullMQ job queues all share the same Redis instance (different key prefixes).
6. **Supabase is used as a dumb PostgreSQL host**: No Supabase SDK, no Supabase Auth, no Supabase Realtime. Only the connection string changes between local Docker and production.
